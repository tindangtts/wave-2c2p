1. Create `src/app/api/mock-payment/topup/route.ts` per D-05:
   - POST handler: accepts `{ amount: number, channel: string }` in body
   - Validates with topupAmountSchema + topupChannelSchema
   - Creates Supabase server client, gets authenticated user
   - Fetches wallet, validates amount <= max_topup
   - Inserts transaction: type="add_money", status="pending", amount in satang, reference_number generated (TOPUP-{timestamp}-{random4})
   - Returns `{ transaction_id, qr_data: { paymentCode: "9300596914", amount: amountInBaht, merchantName: "2C2P PLUS (THAILAND) CO., LTD.", expiresAt: now + 15min }, status: "pending" }`
   - After response, use `setTimeout` with `MOCK_TOPUP_DELAY_MS` env var (default 5000) to update transaction status to "success" and increment wallet balance. Use `after()` from `next/server` if available, otherwise fire-and-forget with the Supabase admin client.

2. Create `src/app/api/mock-payment/withdraw/route.ts` per D-08:
   - POST handler: accepts `{ amount: number, recipient_id: string }`
   - Validates amount > 0, fetches wallet, checks amount <= balance
   - Inserts transaction: type="withdraw", status="pending", recipient_id
   - Decrements wallet balance immediately
   - Returns `{ transaction_id, status: "pending" }`
   - Mock auto-completes: update status to "success" after 2s delay (same pattern as topup)

3. Create `src/app/api/transactions/route.ts`:
   - GET handler with searchParams: page (default 0), limit (default 20), type (optional), status (optional), dateFrom (optional), dateTo (optional)
   - Creates Supabase server client, gets authenticated user
   - Builds query: `supabase.from('transactions').select('*').eq('user_id', user.id).order('created_at', { ascending: false }).range(page*limit, (page+1)*limit - 1)`
   - Apply filters: if type present and not "all", `.eq('type', type)`; if status present and not "all", `.eq('status', status)`; if dateFrom, `.gte('created_at', dateFrom)`; if dateTo, `.lte('created_at', dateTo + 'T23:59:59')`
   - Returns `Transaction[]` array (empty array if no results)
   - Include recipient data: `.select('*, recipients(*)')` if join available, otherwise return recipient_id and let client resolve
