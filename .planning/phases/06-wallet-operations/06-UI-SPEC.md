## Typography

Inherited from Phase 1 Foundation — no new sizes or weights introduced.

| Role | Size | Weight | Line Height | Letter Spacing | Usage in Wallet Operations Phase |
|------|------|--------|-------------|----------------|----------------------------------|
| Display | 28px / 1.75rem | 700 (bold) | 1.2 | -0.02em | Not used |
| Heading | 20px / 1.25rem | 700 (bold) | 1.3 | -0.01em | Screen titles in yellow headers ("Add Money", "Top-up with QR", "Transaction History", "Withdraw"); section headings ("Top-up Channels", "Top-up via Banking Services", "Top-up via 123 Service") |
| Body | 16px / 1rem | 400 (regular) | 1.5 | 0 | Channel names; transaction row recipient name; filter chip labels; balance display value |
| Caption | 12px / 0.75rem | 400 (regular) | 1.4 | 0.01em | "Minimum top-up: 150 THB"; "Max Top-up: 25,000.00"; QR payment code; QR expiry label; date on transaction rows; "Please pay before" text; form helper text |

Two weights only: 700 (bold) for headings and emphasis, 400 (regular) for body and caption.

**Wallet operations-specific typography rules:**

- Amount entry display: 48px weight 700 `#212121` on `#FFFFFF` — matches Phase 5 amount keypad; "THB" suffix in 16px 400
- Balance display line: 14px weight 400 `#757575` — "Wallet Balance: 10,000.00" below amount input
- Max top-up line: 14px weight 400 `#757575` — "Max Top-up: 25,000.00" below balance
- QR payment code: 16px weight 700 `#212121` — "Payment Code: 9300596914"
- QR amount: 20px weight 700 `#212121` — "Amount (THB): 250.00" — financial value needs prominence
- QR merchant name: 14px weight 400 `#757575` — "2C2P PLUS (THAILAND) CO., LTD."
- QR expiry timer: 12px weight 700 `#F44336` when < 60s, `#FF9800` when 1–5 min, `#212121` when > 5 min — "Expires in MM:SS"
- Filter chip label: 14px weight 400 `#212121` (active chip) / `#757575` (inactive chip)
- Transaction date group header: 12px weight 400 `#9E9E9E` — sticky date separator ("Today", "Yesterday", "April 12")
- Transaction amount debit: 16px weight 700 `#F44336` — debit amounts (top-up fees, withdrawals)
- Transaction amount credit: 16px weight 700 `#00C853` — credit amounts (successful top-ups)
- Screen title in yellow header: 20px weight 700 `#0091EA` blue — consistent with app header pattern
- "Minimum top-up: 150 THB": 12px weight 400 `#757575` — below amount input
- Section heading within Add Money screen: 16px weight 700 `#212121` — "Top-up Channels", "Top-up via Banking Services"
- Bank icon button label: 12px weight 400 `#212121` — below bank logo

Source: Phase 1 UI-SPEC typography; Phase 5 UI-SPEC; Pencil design reference (nr3TU, xAphU, HDpqP); 06-CONTEXT.md.

## Interaction Contracts

### Amount Input — Numeric Keypad

Inherited pattern from Phase 5 (amounts in satang integer arithmetic):

| Interaction | Behavior |
|-------------|----------|
| Digit tap | Appends digit; updates display |
| Decimal tap | Adds "."; ignored if already present |
| Backspace tap | Removes last digit |
| Backspace long-press (300ms) | Clears full amount (pointerDown/pointerUp pattern from Phase 5) |
| Amount above maximum | CTA disabled; red caption shown below |
| Amount below minimum | CTA disabled; red caption shown below |

### Filter Chips

| Interaction | Behavior |
|-------------|----------|
| Chip tap | Activates chip (yellow), deactivates previous; "All" chip always resets others |
| Multiple type + status | Filters combine with AND — show transactions matching both |
| Date range + chips | Combine with AND |

### Infinite Scroll

| Interaction | Behavior |
|-------------|----------|
| Scroll near bottom (sentinel in viewport) | Trigger `useSWRInfinite` next page load |
| Loading next page | Skeleton rows (3x) appended below list |
| No more pages | Sentinel removed; no loading indicator |
| Pull-to-refresh | Not implemented in Phase 6 — use filter reset as refresh trigger |

### QR Expiry Timer

| Threshold | Color | Action |
|-----------|-------|--------|
| > 5 min | `#212121` | Normal countdown |
| 1–5 min | `#FF9800` | Warning color |
| < 60s | `#F44336` | Critical color |
| 0s (expired) | `#F44336` | QR dims opacity-40; "Generate New QR" CTA replaces "View Instructions" |

### Date Range Picker

| Interaction | Behavior |
|-------------|----------|
| Trigger tap | Opens Popover with Calendar |
| Calendar day tap | Selects start date; second tap selects end date |
| Buddhist year display | Add 543 to Gregorian year for TH locale display only |
| Reset | Tapping "Clear" in popover resets to "Select date range" state |
| Close | Tap outside popover or after selecting both dates |
