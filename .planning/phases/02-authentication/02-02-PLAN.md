    1. Create `src/app/api/auth/otp/send/route.ts`:
       - POST handler receives `{ phone, countryCode }` JSON body
       - Validate with phoneSchema from src/lib/auth/schemas.ts. Return 400 with error on validation failure.
       - If `MOCK_OTP_AUTO_BYPASS !== 'false'` (mock mode — default): return `{ success: true, mock: true }`
       - If real mode: call `supabase.auth.signInWithOtp({ phone: countryCode + phone })` and return result
       - Always return NextResponse.json

    2. Create `src/app/api/auth/otp/verify/route.ts`:
       - POST handler receives `{ phone, countryCode, token }` JSON body
       - If mock mode and token is NOT '000000': return 401 `{ error: 'Invalid OTP' }`
       - If mock mode and token IS '000000' (or any 6-digit in dev):
         a. Use createAdminClient() to check if user exists: `admin.auth.admin.listUsers()` filtered by phone, or try `admin.auth.admin.getUserByPhone(countryCode + phone)` (catch not found)
         b. If no user: `admin.auth.admin.createUser({ phone: countryCode + phone, phone_confirm: true, password: generateDerivedPassword(phone) })` — password derived from phone + server secret (process.env.MOCK_AUTH_SECRET || 'dev-secret')
         c. Sign in with password to get session: use createServerClient from @supabase/ssr with cookie write access. Call `supabase.auth.signInWithPassword({ phone: countryCode + phone, password: derivedPassword })`
         d. The @supabase/ssr server client automatically writes session cookies to the response
         e. Check user_profiles for this user ID. If no row exists, INSERT minimal row: `{ id: user.id, full_name: '', phone, country_code: countryCode, registration_complete: false, registration_step: 1 }`
         f. Return `{ success: true, isNewUser: !profileExists, registrationComplete: profile?.registration_complete ?? false, registrationStep: profile?.registration_step ?? 1 }`
       - The response must include Set-Cookie headers from the Supabase session

    3. Helper function `generateDerivedPassword(phone: string): string` in the verify route file (not exported):
       - `crypto.createHash('sha256').update(phone + (process.env.MOCK_AUTH_SECRET || 'dev-secret')).digest('hex').slice(0, 32)`
       - This is NOT security-critical — it's a deterministic mock password so the same phone always produces the same password.
