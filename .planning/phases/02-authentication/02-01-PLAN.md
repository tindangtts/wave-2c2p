    - phoneSchema: accepts { countryCode: "+66", phone: "081234567" } (9 digits TH) -> valid
    - phoneSchema: rejects { countryCode: "+66", phone: "08123456" } (8 digits TH) -> error "Enter a valid Thai number (9-10 digits after +66)."
    - phoneSchema: accepts { countryCode: "+95", phone: "09123456789" } (11 digits MM) -> valid
    - phoneSchema: rejects { countryCode: "+95", phone: "091234" } (6 digits MM) -> error "Enter a valid Myanmar number (7-11 digits after +95)."
    - phoneSchema: strips non-digit characters before validation
    - personalInfoSchema: requires firstName (min 1), lastName (min 1), dateOfBirth (DD/MM/YYYY format), nationality (enum: thai/myanmar/other)
    - personalInfoSchema: rejects empty firstName -> "First name is required."
    - idDetailsSchema: requires idType (enum: national_id/passport/work_permit/other), idNumber (min 1), idExpiry (DD/MM/YYYY)
    - idDetailsSchema: rejects empty idNumber -> "ID number is required."
    - hashPasscode("123456") returns string starting with "pbkdf2:"
    - verifyPasscode("123456", hashPasscode("123456")) returns true
    - verifyPasscode("000000", hashPasscode("123456")) returns false
    - verifyPasscode uses timingSafeEqual (no timing side-channel)
  </behavior>
