- Auth guard: same pattern as process-transfer (createClient + getUser)

- Validate sender≠receiver: fetch sender wallet by user.id, fetch receiver wallet by ID — if same user return 400 `{ error: "Cannot transfer to your own wallet" }`

- Generate reference: same `generateReference()` helper pattern (TXN-{timestamp}-{random})
- Deduct sender balance; add to receiver balance (two separate supabase updates)
- Insert transaction row: `type: "transfer"`, `channel: "p2p"`, `status: "pending"`, auto-complete after 2s (same setTimeout pattern as process-transfer)
- Return: `{ success: true, transfer: { id, reference_number, amount, fee: 0, channel: "p2p", created_at } }`
- Respect `MOCK_PAYMENT_FAIL` env var (return failure if set to "true")

Create `src/app/api/mock-payment/refresh-secret-code/route.ts` — POST handler:
- Auth guard (same pattern)
- Accept body: `{ transaction_id: string }`

- Update transaction metadata in supabase OR just return new code (mock — no persistence needed)
- Return: `{ success: true, secret_code: "XXXXXX" }`
