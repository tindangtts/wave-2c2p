## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, star button inline margin |
| sm | 8px | Chip gap in filter row, avatar gap in recipient row |
| md | 16px | Card internal padding, page horizontal padding (`--content-padding-x`) |
| lg | 24px | Section spacing between receipt blocks |
| xl | 32px | Major section breaks, bottom-of-page padding |
| 2xl | 48px | Hero/large spacing |
| 3xl | 64px | Bottom nav clearance (`--bottom-nav-height`) |

Exceptions:
- Touch targets minimum 44×44px (star toggle, copy button, kebab menu, share button) — per WCAG 2.5.5
- Recipient row minimum height: 64px (established pattern in `RecipientRow`)
- Receipt card container: `mx-4 mt-4` (16px horizontal margin, 16px top margin — match existing `TransferReceipt`)
- Secret code chip: 48px height to accommodate copy icon + 6-digit code text comfortably
- Filter tab pills: 32px height × 12px horizontal padding (match `HistoryFilterBar` chip pattern exactly)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (`--background`) | Page backgrounds, receipt card, all white surfaces |
| Secondary (30%) | `#F5F5F5` (`--secondary`) / `#FAFAFA` (`--muted`) | Search input fill, skeleton loading, section row backgrounds |
| Accent (10%) | `#0091EA` (`--accent` / `--color-brand-blue`) | See reserved list below |
| Success | `#00C853` (`--color-success`) | Receipt success icon + "Success!" label |
| Destructive | `#F44336` (`--destructive`) | Fee display in receipt (negative amount), delete confirmation button |

**Accent `#0091EA` reserved for:**
- Favourite star fill + stroke when `is_favorite = true`
- "Create new recipient" link text
- Share button text + icon
- Filter tab pill active background: **exception — active chip uses `#FFE600` (brand-yellow), NOT blue accent** (match `HistoryFilterBar` chip pattern)
- Search input focus ring (`focus:ring-[#0091EA]`)
- Copy button icon in secret code chip

**Brand Yellow `#FFE600` reserved for:**
- Active filter tab pill background ("All" / "Favourites" when selected)
- Recipient avatar circle background (already established pattern)
- Selected recipient row left border highlight + `#FFFDE7` row tint

**Do NOT use accent blue for:** buttons (use yellow primary), backgrounds, borders, or general interactive states.

Text on yellow backgrounds: always `#212121` (dark foreground) — WCAG AA compliant.

Source: `globals.css` full color table, `RecipientRow` and `HistoryFilterBar` established patterns.

## Component Specifications

### 1. P2P Wallet ID Entry Screen (`/transfer/p2p`)

**Layout:** `BackHeader` ("Send to Wallet") + scrollable content area + sticky bottom CTA.

**Elements:**
- Wallet ID text input: full-width, `h-14`, `rounded-xl`, `bg-secondary`, `text-base`, placeholder "Enter wallet ID (e.g. W-123456)" — use existing `<Input>` from shadcn
- OR divider: `text-sm text-[#595959]` centered, horizontal rule on both sides using `border-t border-gray-200`
- Scan QR button: full-width outlined button, `h-12`, `rounded-xl`, border `#0091EA`, text `#0091EA`, icon `ScanLine` (Lucide) left of label "Scan Wallet QR"
- CTA button: full-width `h-14 rounded-xl bg-[#FFE600] text-foreground font-bold text-base`, label "Continue", sticky at bottom with 16px padding

**Validation states:**
- Invalid wallet ID format: red border `border-[#F44336]` + `text-xs text-[#F44336]` error message below input
- Own wallet ID entered: same error styling, message "You cannot send to your own wallet"
- Empty: CTA disabled (`opacity-50 cursor-not-allowed`)

### 2. P2P Amount Entry Screen (`/transfer/p2p/amount`)

**Layout:** Reuse existing `/transfer/amount` screen structure. Use `AmountInput` component.

**Differences from standard amount screen:**
- No `ConversionCard` (THB-to-THB, no conversion)
- No `RateTimer`
- Receiver info displayed as a compact chip at top: avatar circle (yellow, 32px) + recipient wallet ID (`text-sm font-bold`) + "2C2P WAVE Wallet" sub-label (`text-xs text-[#595959]`)
- CTA: "Review Transfer"

### 3. Transfer Confirm Screen Extensions

No new screen — extend existing confirm page to handle `type: 'p2p'` case.

**P2P confirm card additions:**
- "Transfer Type" row: label "Type", value "Wallet Transfer" with `Wallet` Lucide icon (16px, `text-[#0091EA]`)
- Remove "Exchange Rate" and "Converted Amount" rows for P2P type
- Channel row shows "2C2P WAVE (P2P)" instead of bank/agent label

### 4. Transfer Receipt Extensions

Extend `TransferReceipt` component with two new surface areas:

**a. Secret Code chip (cash_pickup channel only):**
- Displayed between "Transfer" section and amount breakdown
- Container: `bg-[#FFF9C4] rounded-xl px-4 py-3 mt-4`
- Label: `text-xs font-bold text-[#595959] uppercase tracking-wide` — "SECRET CODE"
- Code value: `text-2xl font-bold text-foreground tracking-[0.15em] font-mono` — "047532" style
- Copy button: `Copy` Lucide icon, 20px, `text-[#0091EA]`, touch target 44×44px, on tap → `toast.success("Code copied")`
- Refresh button: `RefreshCw` Lucide icon, 20px, `text-[#595959]`, touch target 44×44px, placed to the right of copy button

**b. Export / Download button (all channels):**
- Below existing Share button, same horizontal layout
- `Download` Lucide icon + "Save as Image" label
- Same styling as Share button: `text-[#0091EA] text-base min-h-[44px] px-4`
- Uses `html-to-image` `toPng()` on receipt container ref — dynamic import `{ ssr: false }`
- On success: triggers browser download anchor (`filename: receipt-{transactionId}.png`)
- On error: `toast.error("Could not save image")`

**c. Share button update (all channels):**
- Existing Share button extended: first attempts `navigator.share({ files: [pngBlob], title, text })` (image share on mobile)
- Falls back to text-only share if Files API unsupported
- Falls back to clipboard copy if `navigator.share` unavailable

### 5. Recipient List Filter Tabs

Add tab pill row above the existing search bar.

**Tab row:**
- Container: `flex gap-2 px-4 pt-3 pb-1`
- Two pills: "All" and "Favourites"
- Pill sizing: `h-8 px-3 rounded-full text-xs font-normal`
- Active pill: `bg-[#FFE600] text-foreground`
- Inactive pill: `bg-secondary text-[#595959]`
- Pattern: identical to `HistoryFilterBar` `<Chip>` component — reuse or extract shared `<FilterChip>`

**Filter behaviour:**
- "All" tab: shows existing sections (Favourites at top, then All) — current behaviour preserved
- "Favourites" tab: shows ONLY recipients where `is_favourite = true`; empty state shown if none starred
- Filter state: local `useState` — no URL param needed

**Favourites-only empty state:**
- Centred in remaining space, `py-12 px-8 text-center`
- Heading: `text-base font-bold text-foreground` — "No favourites yet"
- Body: `text-sm text-[#595959]` — "Tap the star next to a recipient to add them here."

### 6. QR Scanner P2P Detection

No visual change to `/scan` page layout. Behavioural addition only:

- Detect wallet QR format: value matches regex `/^W-\d{6,}$/` (or app-defined wallet ID scheme)
- On P2P QR match: navigate to `/transfer/p2p/amount` with wallet ID pre-populated via `p2p-store`
- On non-P2P QR match: existing behaviour preserved

## Interaction Contracts

| Interaction | Behaviour | Feedback |
|-------------|-----------|----------|
| Star toggle (RecipientRow) | Optimistic update via SWR mutate; `PATCH /api/recipients/[id]` | Star animates fill: `#0091EA` on favourite, unfills on un-favourite. On API error: `toast.error("Could not update favourite")` + revert |
| Copy secret code | `navigator.clipboard.writeText(code)` | `toast.success("Code copied")` — Sonner toast, bottom of screen |
| Refresh secret code | `POST /api/mock-payment/refresh-secret-code` | Spinner replaces code value during load; new code fades in |
| Save as Image | `html-to-image` toPng on receipt div ref | Loading spinner on "Save as Image" button during capture; download trigger on success |
| Filter tab switch ("All" / "Favourites") | Instant local state change, no loading | Tab pill background transitions `bg-[#FFE600]` with `transition-colors` |
| P2P wallet ID submit | Navigate to `/transfer/p2p/amount` | No loader needed — local navigation |
| Scan wallet QR | Camera already active; on match auto-navigate | Haptic feedback not available in PWA; no special animation beyond route push |

## Accessibility Contract

- All touch targets: minimum 44×44px (star, copy, refresh, share, save, filter chips)
- Star toggle: `aria-label` dynamically set — "Add {name} to favourites" / "Remove {name} from favourites"
- Secret code chip: `aria-live="polite"` on code value span — announces code change after refresh
- Filter tabs: `aria-pressed` on each pill button
- Copy button: `aria-label="Copy secret code"`
- Refresh button: `aria-label="Refresh secret code"`
- Save image button: `aria-label="Save receipt as image"`
- Receipt container: `id="transfer-receipt-export"` — used as target for `html-to-image` ref
- Error messages: `role="alert"` on inline validation text beneath inputs
- Colour contrast: `#212121` on `#FFE600` background ≥ 8:1 (AAA), `#595959` on white ≥ 5.3:1 (AA)
