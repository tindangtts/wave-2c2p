## Spacing Scale

Inherited from Phase 1 Foundation — no new tokens introduced.

| Token | Value | CSS Variable | Usage in eKYC Phase |
|-------|-------|-------------|---------------------|
| xs | 4px | `--space-1` | Icon-to-text gaps, status badge internal padding |
| sm | 8px | `--space-2` | Camera overlay instruction text margins, document type grid gap |
| md | 16px | `--space-4` | Screen horizontal padding, card internal padding, form field gaps |
| lg | 24px | `--space-6` | Section spacing between capture areas, status card sections |
| xl | 32px | `--space-8` | Top breathing room below headers, major section breaks |
| 2xl | 48px | `--space-12` | Camera overlay top/bottom safe zones |
| 3xl | 64px | `--space-16` | Not used in eKYC screens (no bottom nav) |

**eKYC-specific spacing exceptions:**

| Exception | Value | Justification |
|-----------|-------|---------------|
| Camera overlay guide frame margin | 32px from screen edges | Ensures document card outline is clearly visible with breathing room |
| Document type card minimum height | 64px | Touch target compliance (44px min) + visual weight for selectable option |
| Document type grid gap | 12px | 3-column grid needs tighter gap for mobile; 12px is 3x base unit |
| Capture preview thumbnail | 120x80px (ID ratio 85.6:54) | Matches physical ID card aspect ratio (1.586:1) |
| Face verification circle diameter | 240px | Centered in viewport; large enough for face framing guidance |
| Processing step icon size | 40x40px | Visual weight for status step indicators during verification |
| Status card icon size | 56x56px | Prominent visual indicator for KYC status (matches passcode avatar circle from Phase 2) |
| Re-submission field row height | 56px minimum | Accommodates field label + status icon + action link |

Source: D-03, D-05 in CONTEXT.md. UI-UX-REVIEW Section 2.3 (DocumentUpload, CameraOverlay).

## Navigation & Flow Contract

### eKYC Flow Sequence (per D-10)

```
Document Type → Capture Front → Capture Back → Face Verify → Processing → Status
```

All eKYC screens live under `/(auth)/kyc/*` — no bottom nav during onboarding.

### KYC Screen Header Rules

| Screen | Back behavior | Right slot |
|--------|--------------|------------|
| Document Type Selection | Back to previous page (home or registration) | None |
| Document Front Capture | Back to Document Type | None |
| Document Front Review | Back to Capture (retake) | None |
| Document Back Capture | Back to Front Review | None |
| Document Back Review | Back to Capture (retake) | None |
| Face Verification | Back to Back Review | None |
| Face Review | Back to Capture (retake) | None |
| Processing | No back (locked during verification) | None |
| Status | Back to /home (if approved) | None |
| Re-submission | Back to Status page | None |

### Redirect Logic

| Condition | Redirect |
|-----------|---------|
| KYC approved + auto-redirect | `/home` after 2s (per D-12) |
| KYC approved + button tap | `/home` immediately |
| KYC rejected + "Resubmit" tap | `/(auth)/kyc/resubmit` |
| KYC expired modal + "Update Now" | `/(auth)/kyc/document-type` |
| KYC expired modal + "Update Later" | Dismiss modal, restricted features remain locked |

## Interaction Contracts

### Camera Capture Flow

| Step | Action | Visual Feedback |
|------|--------|----------------|
| 1 | User taps shutter or selects file | Brief white flash overlay (100ms) |
| 2 | Image captured | Navigate to review state with preview |
| 3 | User taps "Use Photo" | Save to component state (base64 per D-04), advance to next step |
| 4 | User taps "Retake" | Return to camera overlay |

### Processing Animation

- 3 steps animate sequentially over ~3 seconds
- Each step: pending icon (gray) -> active icon (yellow + spinner) -> completed icon (green + check)
- Steps auto-advance; no user interaction during processing
- `aria-live="polite"` on step container announces each completion

### Auto-Redirect on Approval (per D-12)

- 2-second delay before redirect using `setTimeout`
- "Continue to Home" button visible immediately as fallback
- `aria-live="assertive"` announces redirect

### KYC Data Storage (per D-11)

- All captures stored in component state as base64 (per D-04)
- On final submission: POST to mock API with all images
- API response determines status (approved/rejected based on `MOCK_KYC_AUTO_APPROVE` env var)
- Status persisted to `kyc_submissions` table in Supabase

### Step Indicator Adaptation

The existing `StepIndicator` component (from Phase 2) needs adaptation:

| Change | Detail |
|--------|--------|
| `totalSteps` prop | Change from `1 | 2 | 3` to `number` type (support 5 steps) |
| `currentStep` prop | Change from `1 | 2 | 3` to `number` type |
| i18n namespace | Add `kyc.stepLabel` alongside existing `auth.register.stepLabel` |
| Camera overlay variant | White dots on dark background (new variant prop: `variant="dark"`) |
