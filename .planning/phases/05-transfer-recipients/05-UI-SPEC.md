## Typography

Inherited from Phase 1 Foundation — no new sizes or weights introduced.

| Role | Size | Weight | Line Height | Letter Spacing | Usage in Transfer Phase |
|------|------|--------|-------------|----------------|------------------------|
| Display | 28px / 1.75rem | 700 (bold) | 1.2 | -0.02em | Not used in transfer screens |
| Heading | 20px / 1.25rem | 700 (bold) | 1.3 | -0.01em | Screen titles ("Recipient", "Confirmation", "Transaction Detail"); section headings ("Receiving Channels") |
| Body | 16px / 1rem | 400 (regular) | 1.5 | 0 | Recipient name; channel name; form field labels; note field placeholder |
| Caption | 12px / 0.75rem | 400 (regular) | 1.4 | 0.01em | Transfer type tag; date on receipt; "1 THB = 133.0 MMK" rate line; form helper text; address lines |

Two weights only: 700 (bold) for headings, 400 (regular) for body and caption. Exception: amount entry uses 700 (bold) at 48px — financial value needs strong prominence; receipt total uses 700 (bold) at 20px.

**Transfer-phase-specific typography rules:**

- Amount entry display: 48px, weight 700, `#212121` on `#FFFFFF` — large numeric keypad-driven input; "THB" suffix in 16px 400
- MMK conversion below amount: 16px, weight 400, `#757575` — "= 133,000.00 MMK" displayed below amount
- Rate line: 12px, weight 400, `#757575` — "1 THB = 133.0 MMK" (caption size)
- Rate lock timer: 12px, weight 700, `#F44336` when < 60s remaining, `#FF9800` when 1–3 min, `#212121` when > 3 min
- Channel card name: 16px, weight 700, `#212121`
- Channel card fee: 12px, weight 400, `#757575`
- Channel card converted amount: 16px, weight 700, `#212121`
- Recipient name: 16px, weight 700, `#212121`
- Recipient type tag: 12px, weight 400, `#757575`
- Confirmation amount (large): 28px, weight 700, `#212121` — top of confirmation screen
- Receipt "Success!": 16px, weight 700, `#00C853` — green success badge label
- Receipt section label (Amount / Fee / Total): 16px, weight 400, `#757575` — left side of breakdown rows
- Receipt section value: 16px, weight 700, `#212121` — right side; Total row uses 20px weight 700
- Form field label: 14px weight 400 `#212121` — outside input, above the field (not inside placeholder)
- Form validation error: 12px, weight 400, `#F44336` — below field on error
- Screen title in yellow header: 20px, weight 700, `#0091EA` blue — consistent with app header pattern

Source: Phase 1 UI-SPEC typography; Pencil design reference; D-09, D-12, D-15 in 05-CONTEXT.md.

## Interaction Contracts

### Transfer Flow Navigation

| Step | Route | Back behavior |
|------|-------|---------------|
| 1. Select recipient | `/transfer/recipient` | Back → `/home` |
| 2. Enter amount | `/transfer/amount` | Back → `/transfer/recipient` |
| 3. Select channel | `/transfer/channel` | Back → `/transfer/amount` |
| 4. Confirm | `/transfer/confirm` | Back → `/transfer/channel` |
| 5. Receipt | `/transfer/receipt` | No back — X closes to `/home`, clears store |

**Zustand store (`useTransferStore`) cleared on:**
- Successful receipt close (tap "Close")
- User navigates from receipt screen with X button
- User aborts by going Home from bottom nav

### Rate Lock Timer

| State | Behavior |
|-------|----------|
| Rate fetched on `/transfer/amount` mount | `validUntil` stored in `useTransferStore` |
| Timer displays on `/transfer/confirm` | MM:SS countdown from `validUntil` |
| Timer > 3 minutes | `#212121` text on `#F5F5F5` badge |
| Timer 1–3 minutes | `#FF9800` text on `#FFF3E0` badge |
| Timer < 60 seconds | `#F44336` text on `#FFEBEE` badge + pulsing animation |
| Timer hits 0 on confirm screen | Rate expired toast + automatically refetch rate; timer resets |
| Timer hits 0 during submit | Re-fetch rate before API call; show "Rate updated. Please confirm again." |

### Amount Input (Numeric Keypad)

| Action | Behavior |
|--------|----------|
| Digit tap | Append digit to amount string |
| Decimal tap | Add "." if not already present |
| Backspace tap | Remove last character |
| Long press backspace | Clear entire amount |
| Amount = "" | Display "0" in gray `#9E9E9E` |
| Amount = "0.XX" | Allow (user building decimal) |
| Leading zeros | Strip leading zeros: "007" → "7" |
| Max digits | 8 digits total (max 25000.00) |
| Conversion update | Debounced 100ms after each keypress |

### Recipient Favorite Toggle

| Action | Behavior |
|--------|----------|
| Tap star icon on recipient row | Toggle `is_favorite` via `PATCH /api/recipients/[id]` |
| Optimistic update | Star fills immediately; API call in background |
| On API error | Revert star state + sonner toast "Could not update favorite" |
| Favorite recipients | Float to "Favorites" section in list |

### Recipient Search

| Property | Value |
|----------|-------|
| Input type | Controlled `<input>` with `onChange` |
| Debounce | 200ms before filtering |
| Filter scope | `name` + `transfer_type` fields (client-side filter on loaded data) |
| No results | Show "No recipients match "{query}"" empty state |
| Clear | X button inside search input when query non-empty |

### Kebab Menu (Recipient Row)

| Action | Result |
|--------|--------|
| Tap "Edit" | Navigate to `/transfer/edit-recipient/[id]` (same form, pre-filled) |
| Tap "Delete" | Open `AlertDialog` with delete confirmation |
| Confirm delete | `DELETE /api/recipients/[id]`, remove from list optimistically |
| Cancel | Close dialog, no change |

### Channel Selection

| Property | Value |
|----------|-------|
| Default selection | None (user must choose) |
| Selection model | Single select — tapping a card selects it, deselects others |
| "Next" enabled | Only when a channel is selected |
| Fee display | Per-channel fee shown in card right section |
| Converted amount | Computed as: `Math.round(amountSatang * rate) / 100` — same integer arithmetic as FOUN-08 |

### Passcode Entry (Bottom Sheet)

| Property | Value |
|----------|-------|
| Keyboard | Native numeric keyboard (`inputMode="numeric"`) |
| Dots display | 6 dots updating as digits typed; `input-otp` component |
| Auto-submit | Submits automatically when 6th digit entered |
| Error animation | CSS `animate-shake` on dot row (3 oscillations, 300ms) |
| Attempt counter | 3 attempts before lockout |
| Lockout duration | 5 minutes (handled server-side, not client timer) |

### Receipt Share

| Property | Value |
|----------|-------|
| API | `navigator.share()` if available (PWA) |
| Share content | Text: "Transfer of {amount} THB to {recipient} completed. Ref: {txn_id}. 2c2p WAVE" |
| Fallback | `navigator.clipboard.writeText()` if share API unavailable + toast "Receipt copied to clipboard" |
| No fallback toast needed for share API usage | Native share sheet handles it |

### Transfer Status Polling

| Property | Value |
|----------|-------|
| Initial state on submit | `pending` — shown immediately (optimistic) |
| Polling | `setInterval` every 2s against `GET /api/mock-payment/status/[id]` |
| Auto-complete | Mock resolves to `completed` after 3s |
| Stop polling | On `completed` or `failed` status |
| Pending display | Spinner + "Processing your transfer..." on `/transfer/receipt` before completion |
| Completed display | Full receipt card revealed (per Screen 6 layout above) |
| Failed display | Error card with `XCircle` 48px `#F44336` + "Transfer failed" + "Contact support" link |
