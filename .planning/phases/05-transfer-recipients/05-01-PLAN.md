1. **Create `src/app/api/recipients/route.ts`** (per D-08):
   - `GET`: Fetch all recipients for authenticated user from Supabase `recipients` table. Use Supabase server client. Order by `is_favorite DESC, full_name ASC`. Return `{ recipients: Recipient[] }`. If no auth session, return 401.
   - `POST`: Create new recipient. Accept JSON body matching recipientFormSchema fields. Compute `full_name = first_name + ' ' + last_name`. Insert into Supabase. Return 201 with `{ recipient: Recipient }`. Validate with Zod server-side (import from `src/lib/transfer/schemas.ts`). On validation error, return 400 with `{ error: string, details: ZodError }`.

2. **Create `src/app/api/recipients/[id]/route.ts`** (per D-08):
   - `PUT`: Update recipient by ID. Same validation as POST. Verify ownership (user_id matches session). Return `{ recipient: Recipient }`. 404 if not found, 403 if not owner.
   - `DELETE`: Delete recipient by ID. Verify ownership. Return `{ success: true }`. 404 if not found.
   - `PATCH`: Toggle favorite. Accept `{ is_favorite: boolean }`. Verify ownership. Return `{ recipient: Recipient }`. This is the lightweight toggle endpoint per D-08.

3. **Create `src/app/api/mock-payment/rate/route.ts`** (per D-03):
   - `GET`: Return `{ rate: number, validUntil: string, from: 'THB', to: 'MMK' }`. Rate from `MOCK_EXCHANGE_RATE` env var (default 133.0 per D-03 — note: this is the cross-border remittance rate, different from the existing exchange-rate endpoint's 58.148 domestic rate). `validUntil` = now + 5 minutes (per D-10). This is a NEW endpoint separate from the existing `/api/mock-payment/exchange-rate` which serves a different purpose.

4. **Create `src/app/api/mock-payment/status/[id]/route.ts`** (per D-16):
   - `GET`: Accept transaction ID param. Use an in-memory Map to track creation times. First call for an ID: store `Date.now()`, return `{ status: 'pending' }`. If elapsed < 1.5s: return `pending`. If elapsed < 3s: return `processing`. If elapsed >= 3s: return `completed` (or `failed` if `MOCK_PAYMENT_FAIL=true`). Return shape: `{ id: string, status: 'pending' | 'processing' | 'completed' | 'failed', updated_at: string }`.

Note on fee schedule: The existing `/api/mock-payment/calculate-fees/route.ts` already has fees but they differ from D-14 (Wave Agent: 10, Wave App: 10, Bank Transfer: 50, Cash Pickup: 30). The existing endpoint uses 15/20 for bank/cash. Per D-14 the CONTEXT decisions override — update the fee values in the existing calculate-fees route to match D-14: wave_agent=10, wave_app=10, bank_transfer=50, cash_pickup=30. Also add this file to the modified list.
