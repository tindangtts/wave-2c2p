<objective>
Replace Supabase write patterns in p2p-transfer and topup routes with Drizzle db.batch() atomic writes.

p2p-transfer: currently updates sender balance, then receiver balance, then inserts a transaction — three separate Supabase calls with manual rollbacks. Replace with two Drizzle selects (sender+receiver fetch) followed by a db.batch([senderUpdate, receiverUpdate, txInsert]) for all three writes atomically.

topup: currently inserts a transaction immediately but defers the balance update inside a setTimeout fire-and-forget. This means the balance update can fail silently. Fix by moving the balance update into an immediate db.batch([walletUpdate, txInsert]) call, removing the deferred pattern.

Purpose: Both routes have data consistency bugs. P2P rollback can fail mid-flight; topup balance update can silently fail after the response is sent. db.batch() fixes both.

Output: Two route files where non-demo write paths use db.batch() with no manual rollback and no deferred balance updates.
</objective>

```typescript
// db.batch() runs all queries in a single Neon HTTP transaction
// All queries must be pre-built (no result from query N used as input to query N+1)
// Returns a tuple matching the query array
const [updateResult, insertResult] = await db.batch([
  db.update(wallets).set({ balance: newBalance, updatedAt: new Date() }).where(eq(wallets.id, walletId)),
  db.insert(transactions).values({ ... }).returning({ id: transactions.id }),
] as const)
```
