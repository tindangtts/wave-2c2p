<objective>
Replace manual Supabase write+rollback patterns in process-transfer and withdraw routes with atomic Drizzle db.batch() calls. Both routes currently update wallet balance then insert a transaction row using separate Supabase calls with a manual rollback on failure — this is not truly atomic.

Purpose: If the transaction insert fails after the wallet balance update, the manual rollback itself can fail, leaving the wallet balance changed but no transaction record. Drizzle's db.batch() wraps both statements in a single Neon HTTP transaction, making them atomic.

Output: Two route files where the non-demo write path uses db.batch([walletUpdate, transactionInsert]) instead of sequential Supabase calls with fallback rollback.
</objective>

CRITICAL: neon-http driver does NOT support db.transaction(). Use db.batch() instead:
```typescript
// db.batch() executes all queries atomically in a single Neon HTTP transaction
// Returns array of results matching query order
const [walletResult, txResult] = await db.batch([
  db.update(wallets).set({ balance: newBalance }).where(eq(wallets.id, walletId)).returning({ id: wallets.id }),
  db.insert(transactions).values({ ... }).returning({ id: transactions.id }),
])
```
