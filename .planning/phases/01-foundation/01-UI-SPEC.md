## Spacing Scale

Base unit: 4px (8-point grid with 4px sub-steps for icon gaps)

| Token | Value | CSS Variable | Usage |
|-------|-------|-------------|-------|
| xs | 4px | `--space-1` | Icon-to-text gap, inline chip padding |
| sm | 8px | `--space-2` | Tight element spacing, input horizontal padding |
| md-tight | 12px | `--space-3` | Card internal gutters |
| md | 16px | `--space-4` | Default section padding, card padding, content horizontal padding |
| md-wide | 20px | `--space-5` | Medium section gaps |
| lg | 24px | `--space-6` | Section dividers, form field vertical gaps |
| xl | 32px | `--space-8` | Major section breaks |
| 2xl | 40px | `--space-10` | Screen top/bottom padding |
| 3xl | 48px | `--space-12` | Hero spacing |
| 4xl | 64px | `--space-16` | Bottom nav height / clearance |

**Exceptions (enforced at component level):**
- Touch targets: minimum 44x44px on all interactive elements. Icon-only buttons use `padding: 10px` with `icon-size: 24px` to reach the 44px target without inflating visual size.
- Bottom nav FAB (Add Money): 56x56px (14rem) to be visually prominent — elevated above bar with `-mt-5` offset.
- OTP digit boxes: minimum 48x48px per box with 8px gap between boxes.
- Country code selector trigger: minimum 44px height — full flag+code area is one tappable zone.
- Form field vertical spacing: minimum 8px gap between consecutive tappable fields.

Source: D-11 in CONTEXT.md, Pitfall 13, UI-UX-REVIEW Section 4.1 Touch Target Compliance.

## Typography

**Font stack (primary — Latin/Thai/English):**
```
'Noto Sans Thai', system-ui, -apple-system, 'Segoe UI', sans-serif
```

**Font stack (Myanmar locale — `lang="my"` only):**
```

```

### Critical Contrast Rules (from D-12 in CONTEXT.md + WCAG audit)

| Combination | Contrast | Status | Rule |
|-------------|----------|--------|------|
| `#212121` on `#FFE600` | 14.7:1 | AAA PASS | REQUIRED for all text on yellow backgrounds |
| White on `#FFE600` | 1.07:1 | CRITICAL FAIL | NEVER use. White text on yellow is invisible. |
| `#212121` on `#FFFFFF` | 16.1:1 | AAA PASS | Standard body text |
| `#757575` on `#FFFFFF` | 4.6:1 | AA PASS | Minimum for secondary text. Use gray-600 (`#757575`) or darker. |
| `#9E9E9E` on `#FFFFFF` | 2.8:1 | FAIL | Never use gray-500 for text on white. |
| `#0091EA` on `#FFE600` | 2.3:1 | FAIL | Never use blue text on yellow backgrounds. |
| White on `#0091EA` | 3.0:1 | Borderline | Acceptable only for large text (18px+) or bold text (14px+ bold). |

## Font Loading Contract

This is a Phase 1 deliverable (FOUN-05). Contract for root layout:

```typescript
// src/app/[locale]/layout.tsx (or root layout.tsx)
import { Noto_Sans_Thai, Noto_Sans_Myanmar_UI } from 'next/font/google'

const notoSansThai = Noto_Sans_Thai({
  subsets: ['thai', 'latin'],
  weight: ['400', '600', '700'],
  display: 'swap',
  variable: '--font-thai',
  preload: true,
})

const notoSansMyanmarUI = Noto_Sans_Myanmar_UI({
  subsets: ['myanmar'],
  weight: ['400', '700'],
  display: 'swap',
  variable: '--font-myanmar',
  preload: true,  // preload even when initial locale is EN/TH — prevents FOUST on switch
})
```

Apply via `globals.css`:
```css
:lang(th) { font-family: var(--font-thai), system-ui, sans-serif; }
:lang(my) { font-family: var(--font-myanmar), system-ui, sans-serif; }
```

The `<html>` element must have `lang` attribute set from the cookie-based locale on every render (D-01 in CONTEXT.md). Setting `lang` on `<html>` enables:
- Correct font selection via CSS `:lang()` selector
- Screen reader locale announcement
- `Intl.DateTimeFormat` locale-correct formatting

**Myanmar Zawgyi guard:** All Myanmar-script text input fields must apply `font-family: var(--font-myanmar)` explicitly via inline style or a dedicated CSS class — this overrides the system font and prevents Zawgyi/Unicode collision (Pitfall 7).
