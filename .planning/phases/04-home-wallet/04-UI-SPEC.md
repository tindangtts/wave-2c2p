## Typography

Inherited from Phase 1 Foundation — no new sizes or weights introduced.

| Role | Size | Weight | Line Height | Letter Spacing | Usage in Home Phase |
|------|------|--------|-------------|----------------|---------------------|
| Display | 28px / 1.75rem | 700 (bold) | 1.2 | -0.02em | Not used in home screens |
| Heading | 20px / 1.25rem | 700 (bold) | 1.3 | -0.01em | "Recent History" section title; "Promotions" section title |
| Body | 16px / 1rem | 400 (regular) | 1.5 | 0 | User greeting "Hello, {first_name}"; transaction labels; promo card titles; quick action labels |
| Caption | 12px / 0.75rem | 400 (regular) | 1.4 | 0.01em | Transaction dates; wallet ID string; "Wallet ID:" label prefix; balance currency label (THB); bottom nav tab labels |

Two weights only: 700 (bold) for headings, 400 (regular) for body and caption. Exception: wallet balance amount uses 700 (bold) at 20px — financial value needs visual prominence without a new size token.

**Home-phase-specific typography rules:**

- User greeting: `--text-body` (16px, 700 bold), `#212121` on `#FFE600` yellow — contrast 14.7:1 AAA PASS
- Balance amount: 20px, weight 700, `#212121` on `#FFE600` — same contrast — this reuses heading size for a non-heading context; acceptable as financial figure prominence
- Balance currency label "THB": `--text-caption` (12px, 400), `#212121` on `#FFE600` — displayed adjacent to balance figure
- Hidden balance: "฿ •••••" — same size/weight as balance amount; bullet character replaces digits (D-02)
- Wallet ID label: `--text-caption` (12px, 400), `#212121` on `#FFE600` — "Wallet ID: WAVE-XXXX-XXXX"
- Quick action label: `--text-caption` (12px, 400), `#212121` — centered below icon circle
- Transaction label: `--text-body` (16px, 400), `#212121` — left-aligned; truncate at 1 line if needed
- Transaction date: `--text-caption` (12px, 400), `#757575` — below transaction label
- Transaction amount: `--text-body` (16px, 700 bold), `#212121` — right-aligned; positive amounts in `#00C853`, negative in `#F44336`
- Section heading: `--text-heading` (20px, 700), `#212121` — "Recent History", used with adjacent icon link
- Bottom nav tab label: `--text-caption` (12px, 400), active: `#0091EA`, inactive: `#757575`

Source: Phase 1 UI-SPEC typography; Pencil design reference; D-02, D-03, D-04 in 04-CONTEXT.md.

## Interaction Contracts

### Balance Show/Hide

| State | UI |
|-------|----|
| Visible (default) | Balance amount in full; eye icon is `Eye` (outline) |
| Hidden | "฿ •••••" replaces digits; eye icon is `EyeOff` |
| Persistence | Preference stored in `localStorage` key `wave_balance_visible` (boolean) |
| Toggle touch target | 44×44px minimum around the eye icon |
| Transition | Instant — no animation (financial data, not decorative) |

### Wallet ID Copy

| Property | Value |
|----------|-------|
| Display format | "WAVE-XXXX-XXXX" (masked — last 8 chars of UUID, per D-03) |
| Copy action | `navigator.clipboard.writeText(fullWalletId)` |
| Feedback | Sonner toast "Wallet ID copied" — 2s auto-dismiss, bottom position (above nav) |
| Fallback | If clipboard API unavailable, no error shown — silent fail |

### Balance SWR Revalidation

| Property | Value |
|----------|-------|
| Initial load | Server Component fetches balance for first paint (no skeleton flash) |
| Client revalidation | SWR `useSWR` on wallet endpoint, `revalidateOnFocus: true`, `dedupingInterval: 30000` (30s) |
| Loading state during revalidation | No visible spinner — stale data shown until fresh data arrives |
| Error state | Sonner toast "Unable to refresh balance" if SWR fetch fails after 3 retries |

### Quick Action Navigation

| Action | Route | Behavior |
|--------|-------|----------|
| Transfer | `/transfer` | Navigate — Phase 5 screen |
| Bills | `/bills` | Navigate to placeholder "Coming Soon" page |
| Referral | `/referral` | Navigate to placeholder "Coming Soon" page |
| Withdrawal | `/withdrawal` | Navigate to placeholder "Coming Soon" page (built in Phase 6) |

### Transaction Row Tap

Transaction rows are read-only in Phase 4. No tap interaction required. Phase 6 (History) adds detail drill-down. Do not add tap handlers now.

### Promo Carousel

| Property | Value |
|----------|-------|
| Library | `embla-carousel-react` (already installed) |
| Behavior | Touch/mouse swipe, no auto-scroll (per D-08) |
| Loop | `false` — stops at last card |
| Slide alignment | `start` (left-aligned) |
| Dot sync | `useEmblaCarousel` selected snap index drives active dot |
| Dot tap | Dots are decorative only — no tap-to-navigate |
