  <action>
    Create `src/app/api/mock-payment/visa-card/route.ts` with a single POST handler:

    1. Auth guard: `createClient()` → `supabase.auth.getUser()` → 401 if no user
    2. Parse body: `{ delivery_address, address_line, amount_satang, fee_satang }` — no zod needed (mock endpoint)
    3. Validate: `delivery_address` must be `'current'` or `'mailing'`; if missing return 400 `{ error: 'delivery_address required' }`
    4. Check `process.env.MOCK_PAYMENT_FAIL === 'true'` — if true return:
       ```json
       { "success": false, "error": "Card payment could not be processed. Please try again later." }
       ```
    5. On success, deduct wallet balance: fetch wallet via `supabase.from('wallets').select('id, balance').eq('user_id', user.id).single()`, check balance >= amount_satang + fee_satang (else 400 `{ error: 'Insufficient balance' }`), then update balance.
    6. Insert transaction: `supabase.from('transactions').insert({ user_id, type: 'card_payment', status: 'success', amount: amount_satang + fee_satang, currency: 'THB', description: 'Visa card request' })`
    7. Return success:
       ```json
       {
         "success": true,
         "card_reference": "VISA-{Date.now()}",
         "delivery_address": "<address_line value>",
         "estimated_delivery": "5-7 business days"
       }
       ```
    8. Wrap entire handler in try/catch → 500 `{ error: 'Internal server error' }`

    Key: Use integer arithmetic throughout (no parseFloat). Fee is 1000 satang (10 THB), card amount is 20000 satang (200 THB) — these are passed by the client, not hardcoded server-side.
  </action>
