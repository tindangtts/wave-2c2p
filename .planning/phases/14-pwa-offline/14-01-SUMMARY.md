## Decisions Made

1. **build:pwa uses `--webpack` flag** — Next.js 16.2.3 defaults to Turbopack for both `next dev` and `next build`. The `@serwist/next` webpack plugin does not run under Turbopack, so `public/sw.js` is only generated when building with `--webpack`. Added `build:pwa` script for production PWA builds.

2. **Serwist disabled in development** — `disable: process.env.NODE_ENV === "development"` prevents the service worker from interfering with hot module reload during `next dev`.

3. **Supabase matcher uses NetworkFirst with networkTimeoutSeconds:0** — Effectively NetworkOnly behavior. This ensures auth tokens are never cached.

4. **Serwist as outermost config wrapper** — `withSerwist(withNextIntl(nextConfig))` ensures Serwist's webpack transforms see the complete final config, avoiding empty precache manifests.

### Auto-fixed Issues

**1. [Rule 1 - Bug] Next.js 16.2.3 uses Turbopack by default for `next build`**
- **Found during:** Task 2 verification
- **Issue:** Plain `next build` in Next.js 16.2.3 runs Turbopack, which does not execute webpack plugins. Serwist's `@serwist/next` is a webpack plugin, so `public/sw.js` was not generated on plain `next build`.
- **Fix:** Added `"build:pwa": "next build --webpack"` to package.json scripts. The `--webpack` flag forces webpack and enables Serwist to run and generate `public/sw.js`.
