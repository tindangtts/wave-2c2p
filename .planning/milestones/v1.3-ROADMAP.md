## Milestone Summary

**Key Decisions:**
- Drizzle ORM chosen over raw Supabase client — type-safe schema, lighter than Prisma
- neon-http adapter for Drizzle (stateless HTTP, ideal for Next.js API routes)
- db.batch() for atomic writes (neon-http doesn't support db.transaction())
- Lazy Proxy singleton for db to prevent build-time crashes when DATABASE_URL absent
- All bigint columns use mode: 'number' for TypeScript compatibility
- isDemoMode branches preserved through Phases 18-21, bulk-removed in Phase 22
- Snake_case aliasing in API responses for UI compatibility (Drizzle returns camelCase)

**Issues Resolved:**
- Drizzle camelCase → snake_case mismatch found during integration audit and fixed
- timestamptz not exported from drizzle-orm — used timestamp({ withTimezone: true }) alias
- Auth routes already clean of demo mode (Plan 22-02 was a no-op)

**Technical Debt Incurred:**
- No Drizzle route unit tests (Nyquist non-compliant for Phase 18)
- 6 API routes still use raw Supabase client (bank-accounts, spending-limits, statement, referral/stats, recipients, mock-kyc)
- E2E tests reference DEMO_MODE in comments and assume demo mode behavior
- Seed SQL requires manual auth.users row creation first
