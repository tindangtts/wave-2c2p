## Critical Issues (2)

### NEW-01: Bottom nav Transfer → stale placeholder

Bottom nav center button routes to `/transfer` which is a Phase 4 placeholder page with hardcoded exchange rate (58.148), no recipient/channel selection, no store wiring. The actual Phase 5 transfer flow at `/transfer/recipient` is unreachable from navigation.

**Fix:** Change `href: "/transfer"` to `href: "/transfer/recipient"` in `bottom-nav.tsx`. Delete or redirect stale `/transfer/page.tsx`.

### NEW-02: Transfer quick action removed (INT-02 regression)

Commit `0a0a9a3` fixed INT-02 by setting `href: "/transfer/recipient"`. Commit `30c2475` (88 min later, icon redesign) rewrote `quick-actions.tsx` with PNG icons and dropped the Transfer action entirely. Current file has 5 actions: bills, referral, withdraw, history, visa-card — no Transfer.

**Fix:** Restore Transfer action in `quick-actions.tsx` with `href: "/transfer/recipient"`.

## High Severity Issues (1)

### NEW-03: Visa Card quick action → 404

`quick-actions.tsx` links to `/visa-card` which doesn't exist. The actual card page is at `/profile/card`.

**Fix:** Change `href: "/visa-card"` to `href: "/profile/card"`.

## Warning Issues (2)

### NEW-04: Stale /withdrawal page

Old stub page at `/withdrawal/page.tsx` still exists alongside working `/withdraw`. Dead-end for deep links.

### NEW-05: Transaction status vocabulary mismatch

DB auto-update sets status to `'success'` but transfer receipt checks for `'completed'`. In-memory status endpoint independently returns `'completed'` so the receipt works, but transaction history will show different vocabulary.
