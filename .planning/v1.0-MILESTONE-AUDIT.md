## Critical Integration Gaps (3)

### INT-01: Registration → KYC not wired
After registration, `create-passcode/page.tsx` routes to `/home` instead of `/kyc/document-type`. New users bypass KYC entirely.

**Fix:** Change `router.push('/home')` to `router.push('/kyc/document-type')` in create-passcode page.

### INT-02: Transfer quick action dead-end
QuickActions links to `/transfer` which is an old prototype stub. Phase 5's full transfer flow at `/transfer/recipient` is unreachable.

**Fix:** Update QuickActions to link `/transfer/recipient` OR redirect `/transfer` → `/transfer/recipient`.

### INT-03: process-transfer doesn't write to DB
Transfer API returns mock success but never deducts wallet balance or inserts transaction. Compare to withdraw API which does both correctly.

**Fix:** Add Supabase operations to `process-transfer/route.ts` matching `withdraw/route.ts` pattern.

## Warning Issues (3)

### INT-04: Withdrawal quick action → Coming Soon stub
QuickActions links `/withdrawal` (stub) instead of `/withdraw` (real flow).

### INT-05: Fee schedule mismatch
Channel page shows D-14 fees (50/30 THB) but process-transfer uses old values (15/20).

### INT-06: Transfer status polling regression
Receipt polls status endpoint which starts at "pending", but process-transfer already returned "processing".
